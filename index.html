How the join works

1. Parse SAP file
Reads every transaction line: customer code, product code, quantity, UOM and Kshs value.
2. Enrich from Google Sheets (live)
Product code → Brand · Category · Litres/Kg (Products sheet)
Customer code → Channel · Type · Location (Customers sheet)
New rows added to either sheet are picked up automatically on next Refresh.
3. All views rebuild
Channel, Product, Region breakdowns recalculate from the enriched data.

Google Sheets connection

Your Sheet acts as the master database for Products, Customers & Targets. The dashboard reads it live — no login needed.

Required column headers in each tab
Products:  PRODUCT CODE  ·  DESCRIPTION  ·  BRAND  ·  CATEGORY  ·  UNIT/LITRE
Customers:  CODE  ·  OUTLET  ·  LOCATION  ·  CHANNEL  ·  TYPE
Targets:  MONTH  ·  CHANNEL  ·  TARGET (Kshs)

Tip: Product codes in your Sheet must match SAP codes (e.g. 44000000013). Customer codes should match SAP without leading zeros (e.g. 100018).

How to publish your Google Sheet

This is a one-time setup. Publishing makes the Sheet readable by the dashboard without any login.

1
Open your Google Sheet → click File in the menu bar → Share → Publish to web
2
In the dialog, set the first dropdown to Entire document and the second to Comma-separated values (.csv)
3
Click Publish and confirm. The Sheet is now publicly readable as CSV — it does not become editable by anyone.
4
Copy the Spreadsheet ID from your browser URL bar:
…/spreadsheets/d/1gBuS6eH8yVU…/edit
Paste it into the Spreadsheet ID field on the left, then click Save & connect.
5
The dashboard will fetch your Products, Customers & Targets. Then upload a SAP file — all breakdowns (channel, region, brand) will populate automatically.

⚠ Still getting "Failed to fetch"?

• The Sheet must be published (step 2–3 above) — sharing the link alone is not enough
• Wait 30 seconds after publishing before clicking connect
• Check that the tab names match exactly (case-sensitive)
• If opening this file from your local disk (file://), some browsers block external requests — serve it via a local server or open it in Chrome
